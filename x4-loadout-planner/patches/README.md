# Patches

Running log of visual/UX fixes, one file per change. Each patch is a
standalone unified diff against the previous state; apply with:

    git apply patches/000X-name.patch

or just read it — they're small and documented inline.

| # | File | Summary |
|---|------|---------|
| 0001 | `0001-toggle-active-border.patch` | Fixed missing left border on the active right-hand button in the `.view-toggle` segmented control (Ships/list toggle, Combat mode toggle, Fitting screen combat-context toggle). |
| 0002 | `0002-text-size-controls.patch` | Added A− / reset / A+ text size controls to the topbar as a view option. Scales the whole app via a `--font-scale` CSS variable, persisted to `localStorage` (`x4lp-font-scale`), range 85%–130% in 5% steps. |
| 0003 | `0003-feat-pare-back-Roles-to-beta-2-starter-roles-cus.patch` | Roles is now beta: trimmed the default preset list to two — **Efficient Haulers** (merged the old Station Hauler + Sector Trader weighting) and **Bombers**. Custom roles and per-role weight tuning still work; added the ability to rename and re-describe *any* role (built-in or custom) from the Roles page, with Reset restoring the original name too. Added a "beta" badge to the nav item and page heading. |
| 0004 | `0004-feat-expand-ship-browser-stat-sorts-mobility-com.patch` | Ship browser "Sort by stat" grew from 5 entries to 16, grouped into General / Mobility / Combat / Mining: boost speed, acceleration, agility, 100km/50km sprint, total sustained DPS, volley damage, weapon range, turret tracking, effective HP, shield regen, mining laser output, and a new composite **Mining efficiency /100km** (cargo × mining output, discounted by round-trip travel time). Also fixes two bugs found along the way: (1) selecting any stat sort was silently reset back to "Sort: name" on the next render, and (2) `miningDPS` was silently always zero for every ship because mining weapons store their yield in `damage`, not `burst`/`sustained` — both bugs predate this session's changes. |
| 0005 | `0005-feat-add-Playable-filter-to-the-faction-dropdown.patch` | Added a **★ Playable** entry to the "All factions" dropdown, implemented as a sentinel in the existing cascading filter logic. ⚠️ Shipped with an under-researched exclusion list (only the M0) — corrected in 0007. |
| 0006 | `0006-fix-usability-accessibility-pass-on-type-scale-r.patch` | Two usability fixes bundled together (both touch `styles.css`): **(1)** a full type-scale legibility pass — contrast was already fine (verified with WCAG math), the real issue was size: raised every sub-12px rule to 12px+, bumped ~10 more 12px reading-content labels to 12.5px, and eased letter-spacing on small uppercase accent-teal headers that were fighting legibility. **(2)** redesigned the comparison table — capped column widths instead of stretching to the full page (4-ship compare: 1368px → 812px), added vertical dividers between columns, and added real alternating-row striping via an explicit component-driven class (CSS `nth-child` striped inconsistently across section-header rows). |

No Kha'ak hull is flyable in vanilla — Ravager and Obliterator included (capture mods note they have no airlocks or cockpit spawn point). Playable filter now excludes 14 ships (242 of 256 shown). Faction-specific views (Xenon, Kha'ak) still list every hull. |

Related but separate: a ready-to-run Windows package (`x4-loadout-planner-windows.zip`, not part of this source zip) with a PowerShell-based local server and a double-click `.bat` launcher, since Vite's module/CORS handling blocks plain `file://` double-click use regardless of how the data loads. |
| 0009 | `0009-fix-remove-explanatory-copy-from-Roles-page-and-.patch` | Removed the two intro paragraphs on the Roles page (beta/weighted-priorities explanation and the rename/rescore instructions), the "Custom roles apply to every ship..." caption, and the "In play:" playstyle line on every role card.
